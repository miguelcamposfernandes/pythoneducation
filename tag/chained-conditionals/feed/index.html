2.3 Conditional Execution

The simplest form of conditional statements is an if statement. It looks something like this:

if a < 0: 
    print("a is negative.")

The boolean expression after the if statement is called the condition. In this case the condition is ‘a < 0′, so translated into English is something like: “If a is less than 0, then…” and we end the if statement with a colon character (:). Afterwards comes the action we want Python to execute.

So if the logical condition is True, then Python executes the indented statement (The Action), if it’s False, so a is greater than 0 the indented statement is skipped.

2.4 Alternative Execution

You just learned how an If statement works, but what if you also want something to happen if the If statement is false? Well, there is where the If-else statement comes in. For better explanation let’s look at an example together:

if x > 0:
    print("X is greater than 0.")
else:
    print("X is less than 0.")

In this example there are exactly 2 possibilities: x being greater or less than 0. Our If statement can either be True or False. If our If statement is true it will normally execute his indented statement but now if it’s false it will execute the Else’s intended statement.

Didn’t get it yet? No problem! Let’s try to understand it with some practice!

2.5 Chained Conditionals

Sometimes there are far more than 2 possibilities so we would need to use what’s called a chained conditional:

    print("Correct! 😃")
elif q1_answer == "b":
    print("Noppp")
else: # Here the Else statement is used to catch errors. If the answer was not A or B then it most be something else. So it is an invalid input.
    print("Invalid Input. You need to answer either with A or B.")

‘elif’Â is an abbreviation for “else if” and can be used the in the same way as an If statement. And there isn’t any limit for elifs statements, you use use as many as you want.

2.6 Nested Conditionals

One conditional can also be nested within another:

if a == b:
    print("a is equal to b")
else:
    if a > b:
        print("a is greater than b")
    else:
        print("a is less than b")

In the example above the first branch is a simple If statement and the second branch contains 2 branches of its own. Although there are 2 simple statements, nested conditionals become harder to read and understand so you should only use them when you really need to.

2.7 Catching Exceptions using Try and Except

2 examples above we learned a method to catch errors using an Else statement, but this can’t always work. For instances if we are developing a program that prompts the user for an integer value and we want to create an Else statement that catch the errors, for that we must create a condition with every single letter and symbol the user could have typed, and that would be very time-consuming. That’s why we use the Try and Except method.
